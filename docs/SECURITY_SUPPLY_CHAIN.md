# Supply Chain Security

Rift uses SLSA-compatible provenance and Sigstore Cosign keyless signing to ensure that every release artifact is verifiable and tamper-evident.

## What is signed

| Artifact | Format | Signed? | Attested? |
|---|---|---|---|
| Container image (`ghcr.io/angelp17/rift`) | OCI image | Cosign keyless (OIDC) | SLSA provenance + SBOM |
| Model bundle (`rift-model-bundle-vX.Y.Z.tar.gz`) | Tarball | Cosign blob signature | N/A |
| SBOM (`rift-sbom.spdx.json`) | SPDX JSON | Attached as image attestation | N/A |

### Model bundle contents

The signed model bundle includes:

- Trained model artifacts (`.rift/runs/`)
- Sector configuration files (`configs/`)
- Governance templates (`docs/templates/`)
- Demo transaction (`demo/sample_transaction.json`)

This ensures that not only code and containers are verifiable, but also the ML artifacts used for fraud decisions.

## How signing works

Rift uses **keyless signing** via GitHub Actions OIDC tokens. No manual secret keys are required.

1. GitHub Actions requests an OIDC token during the release workflow
2. Cosign exchanges the OIDC token for a short-lived signing certificate from Sigstore Fulcio
3. The signature and certificate are stored in the Sigstore transparency log (Rekor)
4. Anyone can verify the signature using the transparency log -- no keys to distribute

## How to verify

### Prerequisites

```bash
# Install Cosign
brew install cosign  # macOS
# or: go install github.com/sigstore/cosign/v2/cmd/cosign@latest

# Install GitHub CLI (for provenance checks)
brew install gh
```

### Verify container image

```bash
cosign verify ghcr.io/angelp17/rift:v1.0.0 \
  --certificate-identity-regexp=".*github.*" \
  --certificate-oidc-issuer="https://token.actions.githubusercontent.com"
```

### Verify SBOM attestation

```bash
cosign verify-attestation ghcr.io/angelp17/rift:v1.0.0 \
  --type spdxjson \
  --certificate-identity-regexp=".*github.*" \
  --certificate-oidc-issuer="https://token.actions.githubusercontent.com"
```

### Verify provenance

```bash
gh attestation verify oci://ghcr.io/angelp17/rift:v1.0.0 --repo AngelP17/Rift
```

### Verify model bundle

```bash
cosign verify-blob \
  --signature rift-model-bundle-v1.0.0.sig \
  --certificate rift-model-bundle-v1.0.0.cert \
  --certificate-identity-regexp=".*github.*" \
  --certificate-oidc-issuer="https://token.actions.githubusercontent.com" \
  rift-model-bundle-v1.0.0.tar.gz
```

### One-command verification

```bash
./scripts/verify_release.sh v1.0.0 --bundle rift-model-bundle-v1.0.0.tar.gz
```

## Why this matters

In regulated fraud detection environments, supply chain trust is critical:

- **Audit compliance**: Regulators can verify that the deployed model artifacts match what was built in CI
- **Tamper detection**: Any modification to the container or model bundle invalidates the signature
- **Provenance**: The build environment, source commit, and workflow are cryptographically linked to the artifact
- **Zero-trust deployment**: Downstream systems can verify artifacts without trusting the artifact publisher's infrastructure

## SLSA Level

Rift targets **SLSA Build Level 2**:

- [x] Build service (GitHub Actions)
- [x] Version-controlled source
- [x] Build-as-code (declarative workflow)
- [x] Provenance generated by the build service
- [x] Signed provenance
